# Architecture Plan

## Decisions

- Keep PHP.
- Keep local branch name `master`.
- Use upstream `dev` as the base because it is newer than upstream `master`.
- All features enabled after fresh install, matching current behavior.
- Add a bot menu item: `Settings -> Container manager`.
- Core services cannot be disabled: `php`, `service`, `ng`, `up`.
- New DB source of truth: SQLite at `/data/vpnbot.sqlite`.
- Legacy import is one-shot only. New runtime code must not read legacy JSON/PHP state as primary storage.

## Goals

- Split `app/bot.php` into small, testable classes.
- Make feature availability explicit through a registry.
- Hide buttons and reject callbacks for disabled features.
- Move mutable app state out of ad hoc JSON files.
- Keep generated config files for daemons that require them.
- Keep install UX close to current `make u`.

## Non-Goals

- No language rewrite.
- No full backward-compatible legacy storage layer.
- No multi-server control plane in the first rewrite.
- No UI redesign beyond required menu changes.

## Core Components

- `Bootstrap\Container`: wires DB, config paths, Telegram API, repositories, feature registry, and modules.
- `Telegram\Router`: maps commands/callbacks to handlers. No business logic.
- `FeatureRegistry`: single source for feature id, services, menu entries, config renderers, health checks, and toggle policy.
- `FeatureManager`: enables/disables features, updates DB, writes compose override, stops/starts services, triggers config render.
- `ComposeManager`: owns `docker-compose.override.yml` generation. No regex YAML edits in handlers.
- `ConfigRenderer`: writes daemon files from DB state, with atomic write.
- `Repositories`: SQLite access only.
- `LegacyImporter`: reads old files once and inserts normalized state.

## Toggle Semantics

- Disable feature:
  - set `features.enabled = 0`
  - regenerate compose override
  - stop/remove affected containers
  - hide menu buttons
  - guard callbacks with "Feature disabled"
- Enable feature:
  - set `features.enabled = 1`
  - render required configs
  - regenerate compose override
  - start affected containers
  - restore menu buttons

## Implementation Order

1. Add Composer autoload and bootstrap while keeping old `Bot` working.
2. Add SQLite connection, migrations, and repository interfaces.
3. Add `FeatureRegistry` and `FeatureManager`; keep old menus but filter buttons through registry.
4. Replace current port/container YAML mutations with `ComposeManager`.
5. Split Telegram routing out of `Bot::action()`.
6. Extract modules one by one: WireGuard, Xray, PAC, AdGuard, OpenConnect, Naive, Shadowsocks, Hysteria, DNSTT, MTProto.
7. Move cron jobs into task classes.
8. Delete obsolete code paths after each module is covered.

## First Useful Milestone

- Bot still works.
- `Container manager` exists.
- `wg`, `wg1`, `xr`, `oc`, `np`, `ss`, `ad`, `tg`, `dnstt`, `hy`, `wp` can be toggled in DB.
- Main menu hides disabled feature buttons.
- Compose override is generated by code, not edited by regex.
